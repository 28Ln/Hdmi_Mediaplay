# BTF 播放器控制接口 - 简化版

> 版本: 1.0  
> 播放器包名: `com.btf.rk3568_hdmi_mediaplay`

---

## 一、接口说明

信发系统只需发送 **一个广播 + 一个JSON**，播放器负责解析执行。

```
信发系统  ──── 广播 + JSON ────→  播放器
         ←─── 状态回调广播 ────
```

---

## 二、发送命令

### 广播 Action
```
com.btf.player.action.COMMAND
```

### 参数
| Key | 类型 | 说明 |
|-----|------|------|
| `command_json` | String | JSON 命令字符串 |

### 发送示例
```kotlin
val json = """{"action":"play_all"}"""

val intent = Intent("com.btf.player.action.COMMAND")
intent.setPackage("com.btf.rk3568_hdmi_mediaplay")
intent.putExtra("command_json", json)
sendBroadcast(intent)
```

---

## 三、JSON 命令格式

### 3.1 播放控制

```json
// 播放全部
{"action": "play_all"}

// 暂停全部
{"action": "pause_all"}

// 停止全部
{"action": "stop_all"}

// 播放指定窗口 (player_index: 0-3)
{"action": "play", "player_index": 0}

// 暂停指定窗口
{"action": "pause", "player_index": 1}

// 停止指定窗口
{"action": "stop", "player_index": 2}
```

### 3.2 布局控制

```json
// 设置布局
{
  "action": "set_layout",
  "layout_mode": "QUAD"
}
```

**layout_mode 可选值:**
| 值 | 说明 | 别名 |
|----|------|------|
| `SINGLE` | 单屏 | - |
| `GRID_1X2` | 左右双屏 | `DUAL_HORIZONTAL` |
| `GRID_2X1` | 上下双屏 | `DUAL_VERTICAL` |
| `GRID_2X2` | 四分屏 | `QUAD` |
| `GRID_1X3` | 横向三分 | `SPLIT_1X3` |
| `GRID_3X1` | 纵向三分 | `SPLIT_3X1` |
| `ROW_1X4` | 横向1x4 | `SPLIT_1X4` |
| `COLUMN_4X1` | 纵向4x1 | `SPLIT_4X1` |
| `PIP` | 画中画(1大3小) | - |

### 3.3 媒体设置

```json
// 设置单个文件
{
  "action": "set_media",
  "player_index": 0,
  "media_path": "/sdcard/media/video.mp4"
}

// 设置多个文件 (轮播)
{
  "action": "set_media",
  "player_index": 1,
  "media_paths": [
    "/sdcard/media/img1.jpg",
    "/sdcard/media/img2.jpg",
    "/sdcard/media/img3.jpg"
  ]
}

// 设置目录 (自动扫描)
{
  "action": "set_media_dir",
  "player_index": 0,
  "directory_path": "/sdcard/media/player1"
}

// 清空指定窗口
{"action": "clear", "player_index": 0}

// 清空全部
{"action": "clear_all"}
```

### 3.4 参数设置

```json
// 设置轮播间隔 (秒)
{"action": "set_interval", "interval": 5}

// 设置音量 (0-100)
{"action": "set_volume", "player_index": 0, "volume": 80}

// 设置静音
{"action": "set_mute", "player_index": 0, "mute": true}

// 设置循环模式 (NONE/SINGLE/ALL)
{"action": "set_loop_mode", "loop_mode": "ALL"}
```

### 3.5 应用控制

```json
// 显示应用 (置于前台)
{"action": "show"}

// 隐藏应用 (置于后台)
{"action": "hide"}

// 查询状态
{"action": "get_status"}
```

### 3.6 一次性配置多个 (批量命令)

```json
{
  "action": "batch",
  "commands": [
    {"action": "set_layout", "layout_mode": "QUAD"},
    {"action": "set_media", "player_index": 0, "media_path": "/sdcard/p1/video.mp4"},
    {"action": "set_media", "player_index": 1, "media_paths": ["/sdcard/p2/1.jpg", "/sdcard/p2/2.jpg"]},
    {"action": "set_media", "player_index": 2, "media_path": "/sdcard/p3/video.mp4"},
    {"action": "set_media", "player_index": 3, "media_path": "/sdcard/p4/video.mp4"},
    {"action": "set_interval", "interval": 5},
    {"action": "play_all"}
  ]
}
```

## 四、状态回调

播放器通过广播返回状态，信发系统注册接收即可。

### 回调 Action
```
com.btf.player.status.CALLBACK
```

### 参数
| Key | 类型 | 说明 |
|-----|------|------|
| `status_json` | String | JSON 状态字符串 |

### 状态 JSON 格式

```json
{
  "type": "playback_changed",
  "timestamp": 1706000000000,
  "player_index": 0,
  "state": "playing"
}
```

```json
{
  "type": "error",
  "timestamp": 1706000000000,
  "error_code": 1003,
  "error_message": "媒体文件不存在"
}
```

```json
{
  "type": "status_response",
  "timestamp": 1706000000000,
  "layout_mode": "QUAD",
  "interval_seconds": 5,
  "players": [
    {"index": 0, "state": "playing", "media_path": "/sdcard/video.mp4"},
    {"index": 1, "state": "playing", "media_path": "/sdcard/img1.jpg", "media_count": 3},
    {"index": 2, "state": "idle"},
    {"index": 3, "state": "paused", "media_path": "/sdcard/video2.mp4"}
  ]
}
```

### 状态类型 (type)

| type | 说明 |
|------|------|
| `playback_changed` | 播放状态变化 |
| `playback_completed` | 播放完成 |
| `layout_changed` | 布局变化 |
| `media_loaded` | 媒体加载完成 |
| `error` | 错误 |
| `status_response` | 状态查询响应 |
| `app_started` | 应用启动 |
| `app_stopping` | 应用停止 |

## 五、完整代码示例

### 信发系统端

```kotlin
class PlayerController(private val context: Context) {
    
    /** 发送命令 */
    fun sendCommand(json: String) {
        val intent = Intent("com.btf.player.action.COMMAND")
        intent.setPackage("com.btf.rk3568_hdmi_mediaplay")
        intent.putExtra("command_json", json)
        context.sendBroadcast(intent)
    }
    
    // ========== 便捷方法 ==========
    
    fun playAll() = sendCommand("""{"action":"play_all"}""")
    
    fun pauseAll() = sendCommand("""{"action":"pause_all"}""")
    
    fun stopAll() = sendCommand("""{"action":"stop_all"}""")
    
    fun setLayout(mode: String) = sendCommand("""{"action":"set_layout","layout_mode":"$mode"}""")
    
    fun setMedia(playerIndex: Int, path: String) = 
        sendCommand("""{"action":"set_media","player_index":$playerIndex,"media_path":"$path"}""")
    
    fun setInterval(seconds: Int) = sendCommand("""{"action":"set_interval","interval":$seconds}""")
    
    fun getStatus() = sendCommand("""{"action":"get_status"}""")
}

// 状态接收器
class PlayerStatusReceiver : BroadcastReceiver() {
    
    var onStatus: ((json: String) -> Unit)? = null
    
    override fun onReceive(context: Context, intent: Intent) {
        val json = intent.getStringExtra("status_json") ?: return
        onStatus?.invoke(json)
    }
}

// 使用
class MyActivity : AppCompatActivity() {
    private val controller by lazy { PlayerController(this) }
    private val receiver = PlayerStatusReceiver()
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        
        // 注册状态接收
        receiver.onStatus = { json ->
            Log.d("Player", "收到状态: $json")
            // 解析 JSON 处理...
        }
        registerReceiver(receiver, IntentFilter("com.btf.player.status.CALLBACK"))
        
        // 发送命令
        controller.setLayout("QUAD")
        controller.setMedia(0, "/sdcard/video.mp4")
        controller.setInterval(5)
        controller.playAll()
    }
    
    override fun onDestroy() {
        super.onDestroy()
        unregisterReceiver(receiver)
    }
}
```

## 六、错误码

| 错误码 | 说明 |
|--------|------|
| 1001 | 无效的播放器索引 |
| 1002 | 无效的布局模式 |
| 1003 | 媒体文件不存在 |
| 1004 | 媒体文件无法读取 |
| 1005 | 不支持的媒体格式 |
| 1006 | 媒体加载失败 |
| 2001 | JSON 解析失败 |
| 2002 | 未知的 action |

---

## 七、总结

| 方向 | Action | 参数 |
|------|--------|------|
| 信发→播放器 | `com.btf.player.action.COMMAND` | `command_json` |
| 播放器→信发 | `com.btf.player.status.CALLBACK` | `status_json` |

就这两个广播，所有逻辑都在 JSON 里。
