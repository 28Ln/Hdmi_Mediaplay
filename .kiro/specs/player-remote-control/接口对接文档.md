# BTF 播放器控制接口 - 对接文档

> 版本: 1.0  
> 播放器包名: `com.btf.rk3568_hdmi_mediaplay`

---

## 一、接口概述

信发系统通过 **广播(Broadcast)** 向播放器发送控制命令，播放器执行后通过 **广播** 返回状态。

```
┌─────────────┐    广播命令     ┌─────────────┐
│   信发系统   │ ─────────────→ │   播放器    │
│  Controller │ ←───────────── │   Player    │
└─────────────┘    状态回调     └─────────────┘
```

---

## 二、快速开始

### 2.1 发送命令示例

```kotlin
// 播放全部
val intent = Intent("com.btf.player.action.PLAY_ALL")
intent.setPackage("com.btf.rk3568_hdmi_mediaplay")
sendBroadcast(intent)
```

### 2.2 接收状态示例

```kotlin
// 注册接收器
val filter = IntentFilter().apply {
    addAction("com.btf.player.status.PLAYBACK_CHANGED")
    addAction("com.btf.player.status.ERROR")
}
registerReceiver(statusReceiver, filter)
```

## 三、控制命令列表

> 所有命令都需要设置 `intent.setPackage("com.btf.rk3568_hdmi_mediaplay")`

### 3.1 播放控制

| 命令 | Action | 参数 | 说明 |
|------|--------|------|------|
| 播放全部 | `com.btf.player.action.PLAY_ALL` | 无 | 启动所有播放器 |
| 暂停全部 | `com.btf.player.action.PAUSE_ALL` | 无 | 暂停所有播放器 |
| 停止全部 | `com.btf.player.action.STOP_ALL` | 无 | 停止所有播放器 |
| 播放指定 | `com.btf.player.action.PLAY` | `player_index`: 0-3 | 播放指定窗口 |
| 暂停指定 | `com.btf.player.action.PAUSE` | `player_index`: 0-3 | 暂停指定窗口 |
| 停止指定 | `com.btf.player.action.STOP` | `player_index`: 0-3 | 停止指定窗口 |

### 3.2 布局控制

| 命令 | Action | 参数 | 说明 |
|------|--------|------|------|
| 设置布局 | `com.btf.player.action.SET_LAYOUT` | `layout_mode`: 见下表 | 切换屏幕布局 |

**布局模式 (layout_mode) 可选值:**

| 值 | 说明 | 窗口数 |
|----|------|--------|
| `SINGLE` | 单屏 | 1 |
| `DUAL_HORIZONTAL` | 左右双屏 | 2 |
| `DUAL_VERTICAL` | 上下双屏 | 2 |
| `QUAD` | 四分屏 | 4 |
| `SPLIT_1X3` | 1+3布局 | 4 |
| `SPLIT_3X1` | 3+1布局 | 4 |
| `SPLIT_1X4` | 横向4屏 | 4 |
| `SPLIT_4X1` | 纵向4屏 | 4 |

### 3.3 媒体设置

| 命令 | Action | 参数 | 说明 |
|------|--------|------|------|
| 设置单个文件 | `com.btf.player.action.SET_MEDIA` | `player_index`: 0-3<br>`media_path`: 文件路径 | 设置单个媒体 |
| 设置多个文件 | `com.btf.player.action.SET_MEDIA` | `player_index`: 0-3<br>`media_paths`: 路径数组 | 设置轮播列表 |
| 设置目录 | `com.btf.player.action.SET_MEDIA_DIR` | `player_index`: 0-3<br>`directory_path`: 目录路径 | 扫描目录所有媒体 |
| 清空指定 | `com.btf.player.action.CLEAR` | `player_index`: 0-3 | 清空指定窗口 |
| 清空全部 | `com.btf.player.action.CLEAR_ALL` | 无 | 清空所有窗口 |

### 3.4 参数设置

| 命令 | Action | 参数 | 说明 |
|------|--------|------|------|
| 轮播间隔 | `com.btf.player.action.SET_INTERVAL` | `interval`: 秒数 | 图片轮播间隔 |
| 设置音量 | `com.btf.player.action.SET_VOLUME` | `player_index`: 0-3<br>`volume`: 0-100 | 设置音量 |
| 设置静音 | `com.btf.player.action.SET_MUTE` | `player_index`: 0-3<br>`mute`: true/false | 静音开关 |
| 循环模式 | `com.btf.player.action.SET_LOOP_MODE` | `loop_mode`: NONE/SINGLE/ALL | 循环播放模式 |

### 3.5 应用控制

| 命令 | Action | 参数 | 说明 |
|------|--------|------|------|
| 显示应用 | `com.btf.player.action.SHOW` | 无 | 将播放器置于前台 |
| 隐藏应用 | `com.btf.player.action.HIDE` | 无 | 将播放器置于后台 |
| 查询状态 | `com.btf.player.action.GET_STATUS` | 无 | 请求完整状态 |

## 四、状态回调列表

> 播放器主动发送，信发系统注册 BroadcastReceiver 接收

| 状态 | Action | 参数 | 触发时机 |
|------|--------|------|----------|
| 播放状态变化 | `com.btf.player.status.PLAYBACK_CHANGED` | `player_index`: 窗口号<br>`state`: playing/paused/stopped<br>`timestamp`: 时间戳 | 播放/暂停/停止时 |
| 播放完成 | `com.btf.player.status.PLAYBACK_COMPLETED` | `player_index`: 窗口号<br>`media_path`: 文件路径<br>`timestamp`: 时间戳 | 单个媒体播放结束 |
| 布局变化 | `com.btf.player.status.LAYOUT_CHANGED` | `layout_mode`: 布局模式<br>`timestamp`: 时间戳 | 布局切换后 |
| 媒体加载完成 | `com.btf.player.status.MEDIA_LOADED` | `player_index`: 窗口号<br>`media_path`: 文件路径<br>`timestamp`: 时间戳 | 媒体加载成功 |
| 错误 | `com.btf.player.status.ERROR` | `error_code`: 错误码<br>`error_message`: 错误信息<br>`timestamp`: 时间戳 | 发生错误时 |
| 状态响应 | `com.btf.player.status.RESPONSE` | `status_json`: JSON字符串<br>`timestamp`: 时间戳 | 响应GET_STATUS |
| 应用启动 | `com.btf.player.status.APP_STARTED` | `timestamp`: 时间戳 | 播放器启动完成 |
| 应用停止 | `com.btf.player.status.APP_STOPPING` | `timestamp`: 时间戳 | 播放器即将退出 |

## 五、完整代码示例

### 5.1 发送控制命令

```kotlin
class PlayerController(private val context: Context) {
    
    private val PLAYER_PACKAGE = "com.btf.rk3568_hdmi_mediaplay"
    
    /** 设置四分屏布局 */
    fun setQuadLayout() {
        sendCommand("com.btf.player.action.SET_LAYOUT") {
            putExtra("layout_mode", "QUAD")
        }
    }
    
    /** 设置播放器媒体文件 */
    fun setMedia(playerIndex: Int, filePath: String) {
        sendCommand("com.btf.player.action.SET_MEDIA") {
            putExtra("player_index", playerIndex)
            putExtra("media_path", filePath)
        }
    }
    
    /** 设置播放器多个文件(轮播) */
    fun setMediaList(playerIndex: Int, filePaths: Array<String>) {
        sendCommand("com.btf.player.action.SET_MEDIA") {
            putExtra("player_index", playerIndex)
            putExtra("media_paths", filePaths)
        }
    }
    
    /** 设置轮播间隔 */
    fun setInterval(seconds: Int) {
        sendCommand("com.btf.player.action.SET_INTERVAL") {
            putExtra("interval", seconds)
        }
    }
    
    /** 播放全部 */
    fun playAll() {
        sendCommand("com.btf.player.action.PLAY_ALL")
    }
    
    /** 暂停全部 */
    fun pauseAll() {
        sendCommand("com.btf.player.action.PAUSE_ALL")
    }
    
    /** 停止全部 */
    fun stopAll() {
        sendCommand("com.btf.player.action.STOP_ALL")
    }
    
    /** 查询状态 */
    fun requestStatus() {
        sendCommand("com.btf.player.action.GET_STATUS")
    }
    
    private fun sendCommand(action: String, extras: Intent.() -> Unit = {}) {
        val intent = Intent(action).apply {
            setPackage(PLAYER_PACKAGE)
            extras()
        }
        context.sendBroadcast(intent)
    }
}
```

### 5.2 接收状态回调

```kotlin
class PlayerStatusReceiver : BroadcastReceiver() {
    
    var onPlaybackChanged: ((playerIndex: Int, state: String) -> Unit)? = null
    var onError: ((code: Int, message: String) -> Unit)? = null
    var onStatusResponse: ((json: String) -> Unit)? = null
    
    override fun onReceive(context: Context, intent: Intent) {
        val timestamp = intent.getLongExtra("timestamp", 0)
        
        when (intent.action) {
            "com.btf.player.status.PLAYBACK_CHANGED" -> {
                val index = intent.getIntExtra("player_index", -1)
                val state = intent.getStringExtra("state") ?: ""
                onPlaybackChanged?.invoke(index, state)
            }
            
            "com.btf.player.status.ERROR" -> {
                val code = intent.getIntExtra("error_code", -1)
                val msg = intent.getStringExtra("error_message") ?: ""
                onError?.invoke(code, msg)
            }
            
            "com.btf.player.status.RESPONSE" -> {
                val json = intent.getStringExtra("status_json") ?: ""
                onStatusResponse?.invoke(json)
            }
        }
    }
    
    companion object {
        fun createIntentFilter() = IntentFilter().apply {
            addAction("com.btf.player.status.PLAYBACK_CHANGED")
            addAction("com.btf.player.status.PLAYBACK_COMPLETED")
            addAction("com.btf.player.status.LAYOUT_CHANGED")
            addAction("com.btf.player.status.MEDIA_LOADED")
            addAction("com.btf.player.status.ERROR")
            addAction("com.btf.player.status.RESPONSE")
            addAction("com.btf.player.status.APP_STARTED")
            addAction("com.btf.player.status.APP_STOPPING")
        }
    }
}

// 使用示例
class MyActivity : AppCompatActivity() {
    private val statusReceiver = PlayerStatusReceiver()
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        
        statusReceiver.onPlaybackChanged = { index, state ->
            Log.d("Player", "窗口$index 状态: $state")
        }
        
        statusReceiver.onError = { code, msg ->
            Log.e("Player", "错误[$code]: $msg")
        }
        
        registerReceiver(statusReceiver, PlayerStatusReceiver.createIntentFilter())
    }
    
    override fun onDestroy() {
        super.onDestroy()
        unregisterReceiver(statusReceiver)
    }
}
```

## 七、错误码

| 错误码 | 说明 |
|--------|------|
| 1001 | 无效的播放器索引 (应为0-3) |
| 1002 | 无效的布局模式 |
| 1003 | 媒体文件不存在 |
| 1004 | 媒体文件无法读取 |
| 1005 | 不支持的媒体格式 |
| 1006 | 媒体加载失败 |

## 八、典型对接流程

```
1. 信发系统启动
      ↓
2. 注册状态接收器 (registerReceiver)
      ↓
3. 等待 APP_STARTED 或主动发送 GET_STATUS
      ↓
4. 设置布局 → SET_LAYOUT
      ↓
5. 设置各窗口媒体 → SET_MEDIA
      ↓
6. 设置轮播间隔 → SET_INTERVAL
      ↓
7. 开始播放 → PLAY_ALL
      ↓
8. 监听状态变化，按需调整
```

---

## 九、注意事项

1. **player_index** 取值 0-3，对应界面上的 P1-P4 窗口
2. **media_path** 必须是完整的绝对路径，如 `/sdcard/Download/video.mp4`
3. 所有命令都是异步的，结果通过状态回调返回
4. 建议在发送命令前先调用 `GET_STATUS` 确认播放器状态
